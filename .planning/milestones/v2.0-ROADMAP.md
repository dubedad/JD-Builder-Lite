# Milestone v2.0: UI Redesign

**Status:** SHIPPED 2026-01-25
**Phases:** 08-A through 08-D
**Total Plans:** 8

## Overview

Redesign the JD Builder Lite UI to mirror the OaSIS website interface, following the swimlane process map with numbered steps. This milestone delivered a complete visual and interaction overhaul while maintaining the core compliance and provenance features.

## Phases

### Phase 08-A: Search Bar Redesign

**Goal**: Redesign Step 1 search interface to position search bar above results with Keyword/Code pill toggle and authoritative sources footnote.
**Depends on**: v1.1 Search Functionality
**Plans**: 1 plan

Plans:
- [x] 08-A-01: Pill toggle, authoritative sources footnote, full stack search type support

**Details:**
- Keyword/Code pill toggle above search input with accent color active state
- Dynamic placeholders per search mode
- Authoritative sources footnote: "ESDC OaSIS 2025 NOC v1.0 / ONET (R) 27.2 Database (USDOL/ETA)"
- Full-stack integration: Frontend → API → OASIS scraper

### Phase 08-B: Results Cards & Grid

**Goal**: Redesign Step 4 occupational profile menu with OaSIS-style cards showing 6 data points, custom grid columns, and custom filters.
**Depends on**: Phase 08-A
**Plans**: 3 plans

Plans:
- [x] 08-B-01: Backend: EnrichedSearchResult model, enhanced parser, API update
- [x] 08-B-02: Card View: OaSIS-style cards HTML/CSS/JS, sort controls
- [x] 08-B-03: Filters: Filter panel with Minor Unit Group, Feeder Mobility, Career Progression

**Details:**

- Lead statement, TEER description, broad category name, matching criteria displayed

### Phase 08-C: Profile Page Tabs

**Goal**: Redesign Step 9 profile page with LLM-driven header elements and Job Header tab structure mapping OaSIS categories to JD elements.
**Depends on**: Phase 08-B
**Plans**: 3 plans

Plans:
- [x] 08-C-01: Backend: LLM endpoints for icon selection and occupation description

- [x] 08-C-03: Tab Navigation: ARIA tabs with content mapping to JD elements

**Details:**

## Milestone Summary

**Key Decisions:**
- SRCH-10: Keyword/Code pill toggle (Rationale: Mimics OASIS UI)
- ENRICH-10: Profile-dependent fields as None (Rationale: Progressive enhancement)
- D-ICON-01: 16 semantic icon categories (Rationale: Balanced NOC coverage)
- D-ICON-02: Temperature 0 for icons, 0.3 for descriptions (Rationale: Deterministic icons)
- D-TAB-01: ARIA tab pattern with automatic activation (Rationale: Standard keyboard expectations)
- SEL-02: CSS ::after tooltips (Rationale: No JS library dependency)
- SEL-03: Single Create button (Rationale: Removes dual-button confusion)

**Issues Resolved:**
- Search type selection now persisted and passed to backend
- Card data extraction working from search HTML without profile fetch
- LLM icon selection validated against allowed list with fallback
- Tab content rendering from profile data with proper category mapping

**Issues Deferred:**
- Profile-dependent card fields (example_titles, mobility_progression) require individual profile fetches
- Grid view skills/abilities/knowledge columns require profile data
- Feeder Group Mobility and Career Progression filters need profile data

**Technical Debt Incurred:**
- Filter UI placeholders for Feeder and Career (functionality deferred)
- Grid view placeholder text for profile-dependent columns
