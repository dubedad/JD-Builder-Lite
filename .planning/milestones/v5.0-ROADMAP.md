# Milestone v5.0: JobForge 2.0 Integration

**Status:** ✅ SHIPPED 2026-03-10

## Overview

Replace live OASIS scraping with JobForge 2.0 gold parquet as the primary data source for search and profile data. OASIS scraping retained as explicit fallback for data not covered by parquet. TBS Directive compliance extended to distinguish parquet-sourced from OASIS-scraped content in provenance metadata.

## Milestone Summary

**Key Decisions:**
- Parquet-first with OASIS fallback: try parquet, fall through transparently on None/LOAD_ERROR/NOT_FOUND
- CoverageStatus(str, Enum) not StrEnum: consistent with EnrichmentSource pattern in codebase
- Column whitespace stripping at read_parquet_safe time: strips once at read, all callers get clean columns
- Two-block try/except in /api/profile: parquet tabs survive OASIS outages; degraded-but-functional, not error
- Auto-detect code search by ^\d{5}$ pattern: defensive, regardless of search_type parameter
- OASIS profile URL fix: /OASIS/OASISOccProfile/{code}.00 — pre-existing breakage discovered and fixed in Phase 23
- Per-section provenance in all 5 JD sections: fully satisfies PROF-03 and TBS Directive compliance requirements
- Symmetric search scoring: OASIS and parquet stem-in-title tier both score 90 — no ranking bias by data source

**Issues Resolved:**
- Silent data load failures in labels_loader.py (4 print() calls, 6 bare except clauses) and vocabulary/index.py (3 paths)
- Pre-existing OASIS profile URL returning 404 (/OaSIS/OaSISSOccProfile → /OASIS/OASISOccProfile with .00 suffix)
- Stem-in-title scoring inconsistency between OASIS (was 85) and parquet (90) search paths
- working_conditions provenance missing from frontend export (PROF-03 gap — closed by Phase 24 TD-1)
- _map_working_conditions_enriched() structural inconsistency vs effort/responsibility mapper pattern

**Issues Deferred (v5.1+ candidates):**
- element_main_duties.parquet ETL gap: 8 rows / 3 profiles (source: 900); OASIS fallback unconditional until JobForge ETL runs
- Abilities/Knowledge tabs render profile.skills?.statements (UI coupling artifact): fix requires splitting backend response into three top-level keys

**Technical Debt Incurred:**
- None introduced this milestone
