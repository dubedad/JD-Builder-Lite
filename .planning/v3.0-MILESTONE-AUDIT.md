## Critical Gaps

### Gap 1: Frontend → Export Wiring (CRITICAL)

**Problem:** When user exports to PDF/DOCX, the styled statement data stored in `container.dataset.styledData` is NOT included in the export request payload.

**Impact:** Users generate styled statements, see them in the UI, then export — but the export shows original statements only, not the styled versions.

**Location:**
- Frontend: Export handlers do not collect styled data from DOM
- Backend: `build_export_data()` in `export_service.py` does not receive/populate `styled_variant`

**Fix Required:**
1. Extend export request to include styled statements array
2. Frontend: Collect `styledData` from all statement containers before export
3. Backend: `build_export_data()` populates `StatementExport.styled_variant`

### Gap 2: build_styled_content_disclosure() Never Called

**Problem:** The function exists in `export_service.py` (lines 192-264) but is never invoked to add the styled content disclosure section to compliance appendix.

**Impact:** Even if Gap 1 is fixed, the compliance appendix won't show vocabulary audit or confidence distribution.

**Location:** `export_service.py` — need to call `build_styled_content_disclosure()` and add to `compliance_sections`
