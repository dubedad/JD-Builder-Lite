# Data Request: v5.0 Job Evaluation Automation

**Project:** JD Builder Lite
**Milestone:** v5.0 - Job Evaluation Standard Application
**Prepared:** 2026-02-04
**Purpose:** Specify data requirements for full classification automation (TBS Step 5)

---

## Overview

To complete automation of the TBS job classification process, we need data for three areas currently missing from the system. This document specifies what's needed, acceptable formats, and how the data will be used.

**Policy Provenance:** Requirements derive from `[JES]` Job Evaluation Standards and `[DOC-B.2.2.1]` Directive on Classification.

---

## 1. Organizational Context Data

**TBS Requirement:** `[DOC-B.2.2.1a]` - Job descriptions must include "organizational context, mandate and supervisor–subordinate relationships"

### What We Need

| Field | Description | Required? | Example |
|-------|-------------|-----------|---------|
| `org_code` | Department/Agency code | Yes | `ESDC`, `TBS`, `DND` |
| `unit_code` | Branch/Division identifier | Yes | `IITB-001`, `HR-CORP` |
| `unit_name` | Branch/Division name | Yes | `Innovation, Information and Technology Branch` |
| `mandate` | Unit mandate statement | Yes | `"Delivers digital services to support..."` |
| `supervisor_position_number` | Supervisor's position # | Preferred | `12345678` |
| `supervisor_group` | Supervisor's classification group | Preferred | `EX`, `CS`, `EC` |
| `supervisor_level` | Supervisor's level | Preferred | `01`, `02`, `03` |
| `parent_unit_code` | Hierarchical parent | Preferred | `IITB` |

### Acceptable Formats

- **CSV/Excel** - Preferred for simple import
- **JSON** - If hierarchical structure needed
- **Database export** - SQL dump or direct connection
- **API** - If real-time sync desired

### Sample Structure (CSV)

```csv
org_code,unit_code,unit_name,mandate,supervisor_position_number,supervisor_group,supervisor_level,parent_unit_code
ESDC,IITB-DDS,Digital Delivery Services,"Delivers enterprise digital platforms and services to ESDC program areas",98765432,EX,02,IITB
ESDC,IITB-ESS,Enterprise Solutions Services,"Provides enterprise application development and support",98765433,EX,02,IITB
```

### How It Will Be Used

- Pre-populate Organizational Context section of job descriptions
- Enable supervisor relationship lookups
- Support organizational hierarchy navigation

## 2. Client-Service Results Data

**TBS Requirement:** `[GUIDE-S2.P1]` - Primary purpose determined through review of work description, particularly Client-Service Results

### What We Need

**Option A: CSR Templates by Classification**

| Field | Description | Required? | Example |
|-------|-------------|-----------|---------|
| `group_code` | Classification group | Yes | `CS`, `EC`, `PM` |
| `level` | Classification level | Yes | `02`, `04`, `05` |
| `csr_template` | Template CSR statement | Yes | `"Provides {service} to {client} to achieve {outcome}"` |
| `typical_clients` | Common client types | Preferred | `"Internal business units, External partners"` |
| `typical_outcomes` | Common outcome types | Preferred | `"Policy advice, System delivery, Program support"` |

**Option B: CSR Statement Library**

| Field | Description | Required? | Example |
|-------|-------------|-----------|---------|
| `csr_id` | Unique identifier | Yes | `CSR-001` |
| `statement` | Full CSR statement | Yes | `"Provides technical expertise to..."` |
| `applicable_groups` | Which groups this applies to | Yes | `["CS", "IT"]` |
| `applicable_levels` | Which levels | Yes | `["02", "03", "04"]` |
| `keywords` | Matching keywords | Preferred | `["technical", "systems", "development"]` |

### Acceptable Formats

- **CSV/Excel** - For flat template lists
- **JSON** - For nested structures with arrays
- **Markdown** - For human-readable templates

### How It Will Be Used

- Generate/suggest CSR statements based on selected NOC profile
- Derive primary purpose for classification matching
- Pre-populate job description Client-Service Results section

## 3. Job Evaluation Standards Data

**TBS Requirement:** `[JES]` - Standards define factors, degrees, and point values for determining classification level

### What We Need

#### 3a. Standards Master List

| Field | Description | Required? | Example |
|-------|-------------|-----------|---------|
| `standard_code` | Standard identifier | Yes | `EC`, `CS`, `PM` |
| `standard_name` | Full name | Yes | `Economics and Social Science Services` |
| `total_points` | Maximum possible points | Yes | `1000` |
| `effective_date` | When standard took effect | Yes | `2017-04-01` |
| `source_url` | TBS source link | Preferred | `https://www.canada.ca/...` |

#### 3b. Evaluation Factors (per standard)

| Field | Description | Required? | Example |
|-------|-------------|-----------|---------|
| `standard_code` | Parent standard | Yes | `EC` |
| `factor_code` | Factor identifier | Yes | `DM` |
| `factor_name` | Full name | Yes | `Decision Making` |
| `criterion` | Statutory criterion | Yes | `responsibility` |
| `max_points` | Maximum points for factor | Yes | `210` |
| `weight_percent` | Weight of total | Yes | `21.0` |
| `definition` | Factor definition text | Yes | `"Measures latitude in decision-making..."` |

#### 3c. Degree Levels (per factor)

| Field | Description | Required? | Example |
|-------|-------------|-----------|---------|
| `standard_code` | Parent standard | Yes | `EC` |
| `factor_code` | Parent factor | Yes | `DM` |
| `degree_number` | Degree level (1-8) | Yes | `5` |
| `points` | Points for this degree | Yes | `90` |
| `description` | Degree description | Yes | `"Multiple areas, departmental programs"` |

#### 3d. Level Ranges (per standard)

| Field | Description | Required? | Example |
|-------|-------------|-----------|---------|
| `standard_code` | Parent standard | Yes | `EC` |
| `level_code` | Full level | Yes | `EC-04` |
| `min_points` | Minimum points | Yes | `250` |
| `max_points` | Maximum points | Yes | `344` |

### Acceptable Formats

- **Excel workbook** - Separate sheets for standards, factors, degrees, levels (Preferred)
- **CSV files** - One per table type
- **JSON** - Nested structure with standards → factors → degrees
- **Structured PDF** - If tables are extractable

### Sample Structure (Excel)

**Sheet 1: Standards**
| standard_code | standard_name | total_points | effective_date |
|---------------|---------------|--------------|----------------|
| EC | Economics and Social Science Services | 1000 | 2017-04-01 |

**Sheet 2: Factors**
| standard_code | factor_code | factor_name | criterion | max_points | weight_percent |
|---------------|-------------|-------------|-----------|------------|----------------|
| EC | DM | Decision Making | responsibility | 210 | 21.0 |
| EC | LO | Leadership and Operations | responsibility | 140 | 14.0 |

**Sheet 3: Degrees**
| standard_code | factor_code | degree_number | points | description |
|---------------|-------------|---------------|--------|-------------|
| EC | DM | 1 | 5 | Issue-specific, own work unit impact |
| EC | DM | 2 | 15 | Issue-specific, project/case components |

**Sheet 4: Levels**
| standard_code | level_code | min_points | max_points |
|---------------|------------|------------|------------|
| EC | EC-01 | 40 | 99 |
| EC | EC-02 | 100 | 169 |

### How It Will Be Used

- Score job descriptions against evaluation factors
- Calculate total points from degree selections
- Determine classification level from point ranges
- Provide transparent rationale for level recommendations

## Data Quality Requirements

| Requirement | Reason |
|-------------|--------|
| **Complete** | All factors/degrees for each standard needed |
| **Consistent** | Points must sum correctly, levels must not overlap |
| **Current** | Use latest effective version of each standard |
| **Sourced** | Include TBS source URL for provenance tracking |
