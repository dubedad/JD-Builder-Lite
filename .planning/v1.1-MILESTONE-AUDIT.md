## Integration Findings

### Critical — IMG-03 Path Mismatch (image display broken)

`fetch_images.py` writes `image_path` values as `functions/slug.jpg` (relative to `static/images/`).
All three browse templates pass this value directly to `url_for('static', path=fn.image_path)`.
FastAPI resolves this to `/static/functions/slug.jpg` — but the file lives at `/static/images/functions/slug.jpg`.

**Result:** Every Unsplash photo produces a 404. All cards fall back to gradient. Users see styled gradient placeholders instead of photos.

**Fix options (either):**
- Template: `url_for('static', path='images/' + fn.image_path)` in all three browse templates
- DB: store `image_path` as `images/functions/slug.jpg` in `fetch_images.py` and update existing rows

### Medium — IMG-02 Partial (families + titles images not fetched)

`fetch_images.py` has been run for function-level images only (22 files confirmed in `static/images/functions/`). The `families/` and `titles/` subdirectories have not been populated. After the IMG-03 path fix is applied, L1 cards will show photos. L2 and L3 cards will still show gradient fallback until the pipeline is run for the remaining 2,221 entities.

### Low — DETAIL-04 Fallback Breadcrumb

`career_detail.html` line 159 else-branch generates `href="/careers/{{ job.family_slug }}"` when function_slug JOIN returns NULL. That URL would match the function-level route and 404. Correct fallback is `href="/careers"`. Unreachable in practice when `migrate_v11.py` has fully run.
