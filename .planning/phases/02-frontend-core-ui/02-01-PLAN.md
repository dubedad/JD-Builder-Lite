Create the API client and wire up search functionality:

**static/js/api.js:**
```javascript
// API client for Flask backend
const api = {
    baseUrl: '/api',

    async search(query) {
        const response = await fetch(`${this.baseUrl}/search?q=${encodeURIComponent(query)}`);
        if (!response.ok) {
            const error = await response.json();
            throw new Error(error.detail || error.error || 'Search failed');
        }
        return response.json();
    },

    async getProfile(code) {
        const response = await fetch(`${this.baseUrl}/profile?code=${encodeURIComponent(code)}`);
        if (!response.ok) {
            const error = await response.json();
            throw new Error(error.detail || error.error || 'Profile fetch failed');
        }
        return response.json();
    }
};
```

**static/js/main.js:**
- DOMContentLoaded event listener wraps all initialization
- Cache DOM references: searchInput, searchButton, searchResults, profileInfo, jdSections, actionBar
- Search handler:
  1. Get query from searchInput.value.trim()
  2. Validate minimum 2 characters
  3. Show loading state (disable button, show "Searching...")
  4. Call api.search(query)
  5. On success: render results to searchResults container, show container
  6. On error: call showError(container, message) - creates inline error div with role="alert"
  7. Restore button state
- Result click handler (event delegation on searchResults):
  1. Get NOC code from clicked item's data-code attribute
  2. Hide search results, show skeleton loading in jdSections
  3. Call api.getProfile(code)
  4. On success:
     - Update profileInfo with title, NOC code, OASIS link, timestamp
     - Show profileInfo
     - Store profile data globally (window.currentProfile)
     - Dispatch custom event 'profile-loaded' with profile data
  5. On error: showError(jdSections, message)
- showError(container, message) function:
  1. Create div with class="error-message", role="alert"
  2. Set textContent to message
  3. Prepend to container
  4. Auto-remove after 5 seconds
- showSkeleton(container) function:
  1. Clear container
  2. Insert 5 skeleton accordion sections (one per JD element)
  3. Each has skeleton header and 3 skeleton text lines
- Bind search button click and Enter key in search input

Do NOT use alert() for errors - always use inline error display.
