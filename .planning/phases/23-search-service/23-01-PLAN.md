**Implementation details:**

1. Build paths: `gold = Path(JOBFORGE_GOLD_PATH)`. Load all three files via `read_parquet_safe()`. If ANY file returns `status == CoverageStatus.LOAD_ERROR`, log a warning and return `None`.

2. Extract DataFrames: `labels_df`, `lead_df`, `titles_df`.

3. For `search_type == "Code"` (query is 5-digit numeric string, validated with `re.match(r'^\d{5}$', query)`):
   - Filter `labels_df` where `unit_group_id == query` (exact string match).
   - If no row matches, return `[]`.
   - Build single `EnrichedSearchResult` with `relevance_score=100`, `match_reason=f"Exact NOC code: {query}"`.
   - Return as single-element list.

4. For `search_type == "Keyword"`:
   - Use the same stemming/normalization helpers that exist in `api.py` (copy the logic into private methods on this class so the two paths score consistently):
     - `_stem_word(word)`: strips `(ers?|ing|tion|ment|ed|ly|s)$` suffix if result >= 3 chars.
     - `_normalize_plural(text)`: normalizes `ies` → `y`, `es` → (drop), `s` → (drop) for last-word plural.
   - Compute `query_lower = query.lower()`, `query_stems = [_stem_word(w) for w in query_lower.split()]`, `query_stem = ' '.join(query_stems)`, `query_norm = _normalize_plural(query_lower)`.
   - Join `lead_df` on `unit_group_id` (left join labels onto lead). Build an in-memory structure; do not iterate row-by-row — use pandas vectorized string ops for speed.
   - Scoring (applied to each row in labels_df):
     - `title_lower = Label.lower()`
     - `title_norm = _normalize_plural(title_lower)`
     - Score 100: `query_norm == title_norm` (near-exact after plural normalization)
     - Score 95: `query_lower in title_lower` (title contains full query string)
     - Score 90: `query_stem in title_lower` OR any individual stem in `query_stems` is found in `title_lower`
     - Score 80: any row in `titles_df` with matching `unit_group_id` has `query_lower` in `Job title text`.lower() OR any query stem in it
     - Score 50: `Lead statement` text (from lead_df join) contains `query_lower` or any query stem
     - If none match, skip this row (do not include score 10 results — the OASIS fallback handles obscure matches).
   - Filter to only rows where a score was assigned. If no rows scored, return `[]`.

5. **Result construction** — for each scored row:
   - `noc_code = str(row["unit_group_id"])`
   - `title = str(row["Label"])`
   - `url = f"{OASIS_BASE_URL}/OaSIS/OaSISProfile?code={noc_code}.00&version={OASIS_VERSION}"`
   - `lead_statement`: look up from `lead_df` where `unit_group_id == noc_code`, get `Lead statement` column value; `None` if not found.
   - `broad_category = int(noc_code[0]) if noc_code else None`
   - `sub_major_group = noc_code[:3] if len(noc_code) >= 3 else None`
   - `unit_group = noc_code[:4] if len(noc_code) >= 4 else None`
   - `relevance_score` and `match_reason` from scoring step.
   - All other `EnrichedSearchResult` fields remain `None`.

6. Sort results by `relevance_score` descending. Return list.
