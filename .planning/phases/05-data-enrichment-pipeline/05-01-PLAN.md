  <action>
    Create csv_loader.py following existing singleton pattern (see mapper.py, parser.py).

    Class: GuideCSVLoader
    - __init__: Initialize empty lookup dicts and stats
    - load(path: Path): Load CSV with encoding='utf-8-sig' (handles Windows BOM)
      - Build _by_element_id dict: element_id -> row dict
      - Build _by_title dict: title.lower() -> row dict (case-insensitive)
      - Set _loaded_at timestamp (datetime.utcnow().isoformat())
      - Log column names on load (warning if expected columns missing)
    - lookup(element_id: str = None, title: str = None) -> Optional[dict]:
      - Try element_id first (primary key)
      - Fallback to title.lower() match
      - Track stats: id_matches, title_fallbacks, missing
      - Return None if no match
    - get_stats() -> dict: Return match statistics
    - get_loaded_at() -> Optional[str]: Return load timestamp
    - get_category_definition(category: str) -> Optional[str]: Return category definition

    **CRITICAL: Category Mapping for get_category_definition**

    The mapper.py will call `get_category_definition()` with JD Element names like:
    - "key_activities"
    - "skills"
    - "effort"
    - "responsibility"
    - "working_conditions"

    But guide.csv uses OASIS category names like:
    - "Work Activities"
    - "Skills"
    - "Work Context"

    Add a CATEGORY_MAPPING constant to translate JD element names to CSV category values:

    ```python
    # Maps JD Element names (used in code) to guide.csv category values
    CATEGORY_MAPPING = {
        # JD Element name -> guide.csv category value
        "key_activities": "Work Activities",
        "work_activities": "Work Activities",
        "skills": "Skills",
        "abilities": "Abilities",
        "knowledge": "Knowledge",
        "effort": "Work Context",           # Work Context contains effort items
        "responsibility": "Work Context",   # Work Context contains responsibility items
        "working_conditions": "Work Context",
        "work_context": "Work Context",
        "personal_attributes": "Personal Attributes",
    }
    ```

    Implement get_category_definition:
    ```python
    def get_category_definition(self, category: str) -> Optional[str]:
        """Get category definition from guide.csv.

        Args:
            category: JD Element name (e.g., 'key_activities', 'skills')

        Returns:
            Category definition string, or None if not found
        """
        # Map JD element name to CSV category
        csv_category = CATEGORY_MAPPING.get(category.lower(), category)

        # Find first row with this category and return its category_definition
        for row in self._by_element_id.values():
            if row.get('category', '').strip() == csv_category:
                return row.get('category_definition', '').strip() or None

        # Fallback: try direct match (in case CSV uses JD names)
        for row in self._by_element_id.values():
            if row.get('category', '').lower().strip() == category.lower():
                return row.get('category_definition', '').strip() or None

        return None
    ```

    Module-level singleton:
    ```python
    from pathlib import Path
    guide_csv = GuideCSVLoader()
    # Load on module import
    _data_path = Path(__file__).parent.parent / 'data' / 'guide.csv'
    if _data_path.exists():
        guide_csv.load(_data_path)
    ```
