  <action>
Create the HTML parser that extracts structured data from OASIS pages using BeautifulSoup.

Create `src/services/parser.py` with class `OASISParser`:

1. Class-level constants:
   - `NOC_CODE_PATTERN = re.compile(r'\d{5}(?:\.\d{2})?')` - matches "21232" or "21232.00"

2. `parse_search_results(self, html: str) -> List[SearchResult]`:
   - Create BeautifulSoup with 'lxml' parser
   - Try primary selector for search results, then fallback
   - For each result row:
     - Find link element, extract href
     - Extract NOC code using regex pattern
     - Extract title from link text
     - Skip rows without valid NOC code
   - Return list of SearchResult models

3. `parse_profile(self, html: str, code: str) -> Dict[str, Any]`:
   - Create BeautifulSoup with 'lxml' parser
   - Return dict with:
     - noc_code: code parameter
     - title: extracted from profile_title selector
     - main_duties: list from _extract_list()
     - work_activities: list from _extract_list()
     - skills: list from _extract_list()
     - abilities: list from _extract_list()
     - knowledge: list from _extract_list()
     - work_context: list from _extract_list()
     - employment_requirements: list from _extract_list()

4. `_extract_title(self, soup: BeautifulSoup) -> str`:
   - Try primary selector, then fallback
   - Return stripped text or empty string

5. `_extract_list(self, soup: BeautifulSoup, element_name: str) -> List[str]`:
   - Get selectors from utils.selectors
   - Try primary, then fallback
   - Return list of stripped text content
   - Filter out empty strings

6. Module-level singleton: `parser = OASISParser()`

<task type="auto">
  <name>Task 3: NOC to JD element mapper</name>
  <files>
    src/services/mapper.py
  </files>
  <action>
Create the mapper that transforms parsed NOC data into JD element structure with full provenance.

Create `src/services/mapper.py` with class `JDMapper`:

1. Class constants for filtering work_context into Effort/Responsibility:
   ```python
   EFFORT_KEYWORDS = ["effort", "physical", "mental", "demand", "strain", "lifting", "standing"]
   RESPONSIBILITY_KEYWORDS = ["responsib", "decision", "supervis", "direct", "manage", "lead", "accountab"]
   ```

2. `to_jd_elements(self, noc_data: Dict[str, Any]) -> Dict[str, Any]`:
   - Build base_url from noc_code for provenance
   - Return dict matching ProfileResponse structure:
     - noc_code, title from input
     - key_activities: from _map_key_activities()
     - skills: from _map_skills()
     - effort: from _map_effort()
     - responsibility: from _map_responsibility()
     - working_conditions: from _map_working_conditions()
     - metadata: SourceMetadata with noc_code, profile_url, scraped_at, version

3. Mapping methods (each returns JDElementData):
   - `_map_key_activities(data, url)`: Combine main_duties + work_activities into NOCStatements
   - `_map_skills(data, url)`: Combine skills + abilities + knowledge into NOCStatements
   - `_map_effort(data, url)`: Filter work_context by EFFORT_KEYWORDS
   - `_map_responsibility(data, url)`: Filter work_context by RESPONSIBILITY_KEYWORDS
   - `_map_working_conditions(data, url)`: All work_context items

4. `_make_statements(self, items: List[str], source_attr: str, url: str) -> List[NOCStatement]`:
   - Helper to create NOCStatement for each item
   - source_attribute formatted nicely (e.g., "Main Duties")

5. Module-level singleton: `mapper = JDMapper()`
