  <action>
    COMPLETELY REPLACE `ps_careers_site/pipeline/fetch_images.py` with a new Unsplash-based async pipeline. Per D-07: do not adapt the old Pexels script — rewrite from scratch.

    **Module structure:**

    1. **Constants and config:**
       - `BASE_DIR = Path(__file__).parent.parent` (ps_careers_site/)
       - `DB_PATH = BASE_DIR / "careers.sqlite"`
       - `IMAGES_DIR = BASE_DIR / "static" / "images"`
       - `PROGRESS_FILE = Path(__file__).parent / "fetch_images_progress.json"`
       - `UNSPLASH_SEARCH_URL = "https://api.unsplash.com/search/photos"`
       - Read `UNSPLASH_ACCESS_KEY` from `os.environ` — raise `RuntimeError("Set UNSPLASH_ACCESS_KEY environment variable")` if missing (per D-03)

    2. **WorkItem dataclass:**
       ```python
       @dataclass
       class WorkItem:
           entity_type: str   # "function", "family", "title"
           slug: str
           name: str
           table: str         # "job_functions", "job_families", "careers"
           pk_col: str        # "job_function_slug", "job_family_slug", "job_title_slug"
           pk_val: str
       ```
       - `key` property: `f"{entity_type}:{slug}"`
       - `dest` property: `IMAGES_DIR / f"{entity_type}s" / f"{slug}.jpg"`
       - `image_rel` property: `f"{entity_type}s/{slug}.jpg"` (relative path for DB storage, per D-02)

    3. **build_query(name: str, entity_type: str) -> str:**
       - Strip parenthetical qualifiers: `re.sub(r"\s*\([^)]*\)", "", name).strip()`
       - function: `f"professionals working in {clean}"`
       - family: `f"person working in {clean}"`
       - title: `f"person as {clean}"`

    4. **fallback_query(name: str) -> str:**
       - `re.sub(r"\s*\([^)]*\)", "", name).strip()`

    5. **search_unsplash(client, query, access_key) -> dict | None:**
       - GET `https://api.unsplash.com/search/photos`
       - Headers: `{"Authorization": f"Client-ID {access_key}"}` (per D-03, header not query param)
       - Params: `query=query, per_page=5, orientation=landscape, content_filter=high`
       - Parse `resp.json().get("results", [])` (NOT "photos" — Unsplash uses "results")
       - Return `results[0]` if results else `None`

    6. **trigger_download(client, download_location, access_key) -> None:**
       - Fire-and-forget GET to `photo["links"]["download_location"]` with auth header (Unsplash ToS requirement)
       - Wrap in try/except — non-fatal

    7. **should_skip(item, done, conn) -> bool:**
       - If `item.key in done`: return True
       - If `item.dest.exists()`: check DB — `SELECT image_path FROM {item.table} WHERE {item.pk_col} = ?`
         - If row exists AND image_path is not None: add to done set, return True
       - Return False
       - Per D-05: BOTH file AND DB must be set to skip; partial state = reprocess

    8. **fetch_and_save(client, sem, item, access_key, conn, done) -> None:**
       - `async with sem:` — bounded concurrency
       - Call search_unsplash with primary query
       - If no results: retry with fallback_query
       - If still no results: log warning, mark done (no infinite retry), save progress, return
       - Call trigger_download (ToS)
       - Download image: `async with client.stream("GET", photo["urls"]["regular"]) as r:` — stream to bytes
       - Save file: `await asyncio.to_thread(item.dest.parent.mkdir, parents=True, exist_ok=True)` then `await asyncio.to_thread(item.dest.write_bytes, data)`
       - Update DB: `conn.execute(f"UPDATE {item.table} SET image_path = ? WHERE {item.pk_col} = ?", (item.image_rel, item.pk_val))` then `conn.commit()`
       - Add to done set, save progress
       - Handle 429: if `resp.status_code == 429`, log `Retry-After` header value, do NOT mark as done, return (will retry on next run)

    9. **build_work_list(conn) -> list[WorkItem]:**
       - Query `job_functions` table (NOT careers DISTINCT): `SELECT job_function_slug, job_function FROM job_functions ORDER BY job_function`
       - Query `job_families` table: `SELECT job_family_slug, job_family FROM job_families ORDER BY job_family`
       - Query `careers` table: `SELECT job_title_slug, job_title FROM careers ORDER BY job_title`
       - Return list in order: functions first, then families, then titles (per D-06)

    10. **load_progress() / save_progress(done):** Same JSON pattern as old script.

    11. **async def run_pipeline(entity_type="all", workers=5):**
        - Load access key from env
        - Connect to DB
        - Build work list, filter by entity_type if not "all"
        - Load progress
        - Filter to items not already done via should_skip
        - `sem = asyncio.Semaphore(workers)`
        - `async with httpx.AsyncClient(timeout=30.0) as client:`
        - `tasks = [asyncio.create_task(fetch_and_save(client, sem, item, ...)) for item in work]`
        - `results = await asyncio.gather(*tasks, return_exceptions=True)`
        - Log any exceptions in results
        - Close DB

    12. **main() with argparse:**
        - `--type`: choices=["function", "family", "title", "all"], default="all"
        - `--workers`: int, default=5
        - `--limit`: int, default=None (process only first N items — for testing with Demo key)
        - `--dry-run`: flag, print work list without fetching
        - Call `asyncio.run(run_pipeline(...))`

    **Critical implementation notes:**
    - Use `httpx.AsyncClient` (already installed 0.27.2) — NOT requests, NOT aiohttp
    - Use `asyncio.Semaphore(workers)` for concurrency — NOT ThreadPoolExecutor
    - Use `asyncio.to_thread()` for blocking file I/O — NOT direct open().write() in async
    - Use `asyncio.gather(*tasks, return_exceptions=True)` — NOT bare gather (swallows exceptions)
    - Store relative path in DB: `"functions/slug.jpg"` NOT absolute path (per D-02)
    - Progress file write AFTER DB commit, not before (per Pitfall 3 from research)
