The filter hierarchy needs sub-major group names and minor group (unit group) names in search results. Currently, search results include `broad_category_name` and `minor_group` (3-digit code) but NOT the textual names for sub-major groups.

The NOC hierarchy structure is already parsed from profile HTML in parser.py `_parse_hierarchy_names()` (line ~250), but search results don't have this data because search HTML doesn't include it. We need to derive hierarchy names from the NOC code structure.

**Approach:** The NOC 2021 structure follows a fixed digit pattern:
- Broad category = 1st digit (already in results as broad_category, broad_category_name)
- TEER = 2nd digit
- Sub-major group = 1st 3 digits (this is what NOC calls "minor group" confusingly; the app calls it minor_group)
- Minor group = 1st 4 digits (this is what NOC calls "unit group")
- Unit group = all 5 digits

The search results already have `minor_group` (3-digit code). We need to add hierarchy group names. Since we don't have a lookup table pre-loaded, we need to build one from the OASIS data we already have, OR use the existing broad_category_name and derive sub-group labels from the results themselves.

**Practical approach (no external data dependency):** Build the hierarchy dynamically from search results:

1. Add fields to `EnrichedSearchResult` model in noc.py:
```python
# In EnrichedSearchResult class, add:
sub_major_group: Optional[str] = None      # First 3 digits of NOC code
sub_major_group_name: Optional[str] = None  # Derived from OASIS data or fallback
unit_group: Optional[str] = None           # First 4 digits
unit_group_name: Optional[str] = None      # Derived from OASIS data or fallback
```

2. In the search route in api.py (the `/api/search` endpoint), after `results = parser.parse_search_results_enhanced(html)`, populate the hierarchy fields from the NOC code:
```python
for r in results:
    code = r.noc_code.split('.')[0]  # Remove decimal suffix
    r.sub_major_group = code[:3] if len(code) >= 3 else None
    r.unit_group = code[:4] if len(code) >= 4 else None
```

3. Group results by sub_major_group code and use the first result's `broad_category_name` + code prefix as fallback name. The OASIS search HTML may contain additional hierarchy info in the result cards -- check `parse_search_results_enhanced()` for any hierarchy data already being extracted.

4. If OASIS search results don't include sub-major group names (they likely don't), use a static mapping. The broad category names are already in `BROAD_CATEGORIES` dict in noc.py. For sub-major groups, we can derive a reasonable label: `"{broad_category_name} ({sub_major_code})"` as a fallback. But ideally, load the NOC 2021 structure.

**Better approach -- use NOC structure CSV:** Check if `data/guide_oasis.csv` or any other data file contains NOC structure hierarchy names. If so, build a lookup. If not, use the code-based grouping in the frontend (group by 3-digit code, show broad category name as parent, individual results under it).

**Final recommendation:** Since the hierarchy data source (dim_noc_structure) is not available in the project, build the hierarchy client-side from search results:
- Sub-major group = group by first 3 digits of NOC code
- Use the `broad_category_name` of the first result in each group as the parent heading
- Minor groups = group by first 4 digits within each sub-major group
- Label = `"{broad_category_name} - {3-digit code}"` for sub-major, result title for minor items

Add `sub_major_group` and `unit_group` fields to EnrichedSearchResult. Populate them from the NOC code in the search route. The frontend will build the hierarchy from these codes.

**A. Update updateFilterOptions() in filters.js:**

Build hierarchy from search results:
```javascript
function updateFilterOptions(results) {
    allResults = results;

    // Build 2-level hierarchy: broad_category_name -> sub_major_group -> results
    const hierarchy = new Map(); // sub_major_group_code -> { name, broadCat, children: Map<unit_group, {name, count}> }

    results.forEach(r => {
        const subMajorCode = r.sub_major_group || (r.noc_code ? r.noc_code.substring(0, 3) : null);
        const unitGroupCode = r.unit_group || (r.noc_code ? r.noc_code.substring(0, 4) : null);
        if (!subMajorCode) return;

        if (!hierarchy.has(subMajorCode)) {
            hierarchy.set(subMajorCode, {
                code: subMajorCode,
                broadCat: r.broad_category_name || 'Unknown',
                children: new Map(),
                totalCount: 0
            });
        }

        const parent = hierarchy.get(subMajorCode);
        parent.totalCount++;

        if (unitGroupCode) {
            if (!parent.children.has(unitGroupCode)) {
                parent.children.set(unitGroupCode, {
                    code: unitGroupCode,
                    name: unitGroupCode,  // Will be overridden with title if available
                    count: 0,
                    results: []
                });
            }
            const child = parent.children.get(unitGroupCode);
            child.count++;
            // Use first result's title as representative name for this unit group
            if (child.count === 1) {
                child.name = r.title || unitGroupCode;
            }
        }
    });

    // Render hierarchical checkboxes
    if (minorGroupOptions) {
        if (hierarchy.size === 0) {
            minorGroupOptions.innerHTML = '<p class="filter-empty">No categories in results</p>';
        } else {
            // Sort parents by broad category name then code
            const sortedParents = Array.from(hierarchy.values())
                .sort((a, b) => a.broadCat.localeCompare(b.broadCat) || a.code.localeCompare(b.code));

            minorGroupOptions.innerHTML = sortedParents.map(parent => {
                const children = Array.from(parent.children.values())
                    .sort((a, b) => a.code.localeCompare(b.code));

                const childrenHtml = children.map(child => `
                    <label class="filter-checkbox filter-checkbox--child">
                        <input type="checkbox"
                               class="child-checkbox"
                               name="minorGroup"
                               value="${escapeHtml(child.code)}"
                               data-parent="${escapeHtml(parent.code)}"
                               ${filters.minorGroup.has(child.code) ? 'checked' : ''}>
                        <span class="filter-checkbox-label">
                            ${escapeHtml(child.name)}
                            <span class="filter-checkbox-count">(${child.count})</span>
                        </span>
                    </label>
                `).join('');

                // Check parent state based on children
                const allChecked = children.every(c => filters.minorGroup.has(c.code));
                const someChecked = children.some(c => filters.minorGroup.has(c.code));

                return `
                    <div class="filter-group">
                        <label class="filter-checkbox filter-checkbox--parent">
                            <input type="checkbox"
                                   class="parent-checkbox"
                                   data-group="${escapeHtml(parent.code)}"
                                   ${allChecked && children.length > 0 ? 'checked' : ''}>
                            <span class="filter-checkbox-label">
                                <strong>${escapeHtml(parent.broadCat)}</strong>
                                <span class="filter-checkbox-code">(${parent.code})</span>
                                <span class="filter-checkbox-count">${parent.totalCount}</span>
                            </span>
                        </label>
                        <div class="filter-group__children">
                            ${childrenHtml}
                        </div>
                    </div>
                `;
            }).join('');

            // Set indeterminate state on parents after rendering
            setTimeout(() => updateParentStates(), 0);
        }
    }

    // Keep feeder/progression placeholders as-is
    if (feederOptions) {
        feederOptions.innerHTML = '<p class="filter-empty">Select a profile to enable mobility filtering</p>';
    }
    if (progressionOptions) {
        progressionOptions.innerHTML = '<p class="filter-empty">Select a profile to enable progression filtering</p>';
    }

    updateClearButton();
}
```

**B. Add parent/child checkbox handlers:**

```javascript
function handleParentCheckboxChange(parentCheckbox) {
    const groupCode = parentCheckbox.dataset.group;
    const children = document.querySelectorAll(
        `input.child-checkbox[data-parent="${groupCode}"]`
    );

    children.forEach(child => {
        child.checked = parentCheckbox.checked;
        if (parentCheckbox.checked) {
            filters.minorGroup.add(child.value);
        } else {
            filters.minorGroup.delete(child.value);
        }
    });

    parentCheckbox.indeterminate = false;
    applyFilters();
    updateClearButton();
}

function updateParentStates() {
    document.querySelectorAll('input.parent-checkbox').forEach(parent => {
        const groupCode = parent.dataset.group;
        const children = document.querySelectorAll(
            `input.child-checkbox[data-parent="${groupCode}"]`
        );
        const total = children.length;
        const checked = Array.from(children).filter(c => c.checked).length;

        parent.checked = checked === total && total > 0;
        parent.indeterminate = checked > 0 && checked < total;
    });
}
```

**C. Update handleFilterChange()** to handle parent vs child checkboxes:

```javascript
function handleFilterChange(e) {
    const checkbox = e.target;
    if (checkbox.type !== 'checkbox') return;

    // Handle parent checkbox
    if (checkbox.classList.contains('parent-checkbox')) {
        handleParentCheckboxChange(checkbox);
        return;
    }

    // Handle child checkbox
    if (checkbox.classList.contains('child-checkbox')) {
        const value = checkbox.value;
        if (checkbox.checked) {
            filters.minorGroup.add(value);
        } else {
            filters.minorGroup.delete(value);
        }
        updateParentStates();
        applyFilters();
        updateClearButton();
        return;
    }

    // Legacy: handle non-hierarchical checkboxes
    const filterType = checkbox.name;
    const value = checkbox.value;
    if (!filters[filterType]) return;

    if (checkbox.checked) {
        filters[filterType].add(value);
    } else {
        filters[filterType].delete(value);
    }
    applyFilters();
    updateClearButton();
}
```

**D. Update applyFilters()** to filter by unit_group code (4-digit) instead of broad_category_name:

```javascript
function applyFilters() {
    const filtered = allResults.filter(result => {
        if (filters.minorGroup.size > 0) {
            // Filter by unit_group (4-digit) code matching checked children
            const unitGroup = result.unit_group || (result.noc_code ? result.noc_code.substring(0, 4) : null);
            if (!unitGroup || !filters.minorGroup.has(unitGroup)) {
                return false;
            }
        }
        return true;
    });

    if (renderCallback) {
        renderCallback(filtered);
    }
}
```
