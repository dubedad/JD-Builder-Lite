  <action>
Extend `ingest.py` with the full ingest implementation:

**`ingest(xlsx_path, db_path)` function:**

1. Open the workbook with openpyxl: `load_workbook(xlsx_path, read_only=True, data_only=True)`. Sheet name: `DRAFT_JA_EN-FR`. Read header row to build a column-index map (handle variations in capitalization by normalizing header values).

2. **Slug collision detection per job family:**
   - First pass: collect all (jt_id, job_title, job_family) tuples from the sheet
   - Build `family_slug_counts: dict[str, Counter]` — for each family, count how many titles produce the same base slug
   - Any slug that appears more than once gets the jt_id suffix appended: `make_slug(job_title, suffix=str(jt_id))`

3. **Row loop:** For each data row, build a dict with these fields:
   - `jt_id`: int(JT_ID)
   - `job_title`: Job_Title
   - `titre_de_poste`: Titre_de_poste (may be None)
   - `job_function`: Job_Function
   - `job_family`: Job_Family
   - `managerial_level`: Managerial_Level
   - `noc_2021_uid`: str(2021_NOC_UID) if not None
   - `noc_2021_title`: 2021_NOC_Title
   - `digital`: Digital
   - `digital_category`: Digital_Category
   - `job_title_slug`: slug (with collision suffix if needed)
   - `job_family_slug`: make_slug(job_family) — same for all rows in same family
   - `card_image_key`: CARD_IMAGE_MAP.get(job_family)  — None if family not in map

4. **Upsert with content preservation** — use `INSERT OR REPLACE` is NOT safe here because it destroys enriched content. Instead use:
   ```sql
   INSERT INTO careers (jt_id, job_title, titre_de_poste, job_function, job_family,
       managerial_level, noc_2021_uid, noc_2021_title, digital, digital_category,
       job_title_slug, job_family_slug, card_image_key, content_status)
   VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, 'empty')
   ON CONFLICT(jt_id) DO UPDATE SET
       job_title       = excluded.job_title,
       titre_de_poste  = excluded.titre_de_poste,
       job_function    = excluded.job_function,
       job_family      = excluded.job_family,
       managerial_level= excluded.managerial_level,
       noc_2021_uid    = excluded.noc_2021_uid,
       noc_2021_title  = excluded.noc_2021_title,
       digital         = excluded.digital,
       digital_category= excluded.digital_category,
       job_title_slug  = excluded.job_title_slug,
       job_family_slug = excluded.job_family_slug,
       card_image_key  = excluded.card_image_key
       -- NOTE: overview, training, entry_plans, part_time, caf_related,
       --       content_status, enriched_at are intentionally NOT updated
       --       so Phase 2 enrichment is preserved on re-run
   ```
   SQLite 3.24+ supports ON CONFLICT DO UPDATE (UPSERT). Require sqlite3 >= 3.24 and raise a clear error if not met.

5. **Summary output** after commit:
   ```
   Ingest complete
   Rows inserted/updated : 1989
   Unique job families   : 209
   Unique job functions  : 22
   Slug collisions fixed : 0
   WARNING: Row count 1989 matches expected 1989
   ```
   If row count != 1,989, print: `WARNING: Expected 1989 rows, got {N}`.
   List each slug collision as: `  COLLISION: "{title}" -> "{slug}-{jt_id}" (jt_id={jt_id})`.

6. **`if __name__ == "__main__"` block:** parse args, call `create_schema(conn)`, call `ingest(xlsx_path, db_path)`.

**Error handling:** Skip rows where JT_ID is None or non-integer (print a warning). Wrap the entire run in try/finally to ensure the db connection is closed.

**Note on CONTEXT.md decision:** The CONTEXT.md specifies `INSERT OR REPLACE` but notes enriched content must be preserved. Use the `ON CONFLICT DO UPDATE` pattern above instead — it satisfies the idempotency requirement while correctly preserving enriched content. This is the correct SQLite UPSERT.
