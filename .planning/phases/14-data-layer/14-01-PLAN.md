Create `src/storage/schema.sql` with the following tables (from RESEARCH.md verified patterns):

1. **scrape_provenance** - HTTP-level metadata for DADM compliance:
   - id (INTEGER PRIMARY KEY)
   - url (TEXT NOT NULL)
   - scraped_at (TEXT NOT NULL) - ISO 8601 UTC
   - http_status (INTEGER NOT NULL)
   - http_headers (TEXT) - JSON string
   - content_hash (TEXT NOT NULL) - SHA-256
   - archive_path (TEXT NOT NULL) - Path to saved HTML
   - parser_version (TEXT NOT NULL)

2. **dim_occupational_group** - Master table with append-only temporal:
   - id (INTEGER PRIMARY KEY)
   - group_code (TEXT NOT NULL) - e.g., "AI", "CP"
   - subgroup (TEXT) - e.g., "Non-Operational" for AI
   - definition (TEXT NOT NULL)
   - qualification_standard_url (TEXT)
   - rates_of_pay_represented_url (TEXT)
   - rates_of_pay_unrepresented_url (TEXT)
   - effective_from (TEXT NOT NULL) - ISO 8601 UTC
   - effective_to (TEXT) - NULL = currently valid
   - source_provenance_id (INTEGER NOT NULL) - FK to scrape_provenance
   - UNIQUE(group_code, subgroup, effective_from)
   - FOREIGN KEY to scrape_provenance

3. **dim_occupational_inclusion** - Child table for inclusion statements:
   - id (INTEGER PRIMARY KEY)
   - group_id (INTEGER NOT NULL) - FK to dim_occupational_group
   - statement (TEXT NOT NULL)
   - order_num (INTEGER NOT NULL) - Original list position
   - paragraph_label (TEXT) - e.g., "I1", "I2" for provenance
   - source_provenance_id (INTEGER NOT NULL)
   - FOREIGN KEY constraints

4. **dim_occupational_exclusion** - Child table for exclusion statements:
   - Same structure as inclusions

5. **verification_event** - Human-in-loop tracking for DADM:
   - id (INTEGER PRIMARY KEY)
   - group_id (INTEGER NOT NULL)
   - verified_at (TEXT NOT NULL)
   - verified_by (TEXT NOT NULL)
   - verification_notes (TEXT)
   - FOREIGN KEY to group

6. **table_of_concordance** - Links groups to evaluation standards (DATA-05):
   - id (INTEGER PRIMARY KEY)
   - group_code (TEXT NOT NULL)
   - job_evaluation_standard_url (TEXT)
   - effective_from (TEXT NOT NULL)
   - effective_to (TEXT)
   - source_provenance_id (INTEGER NOT NULL)

Include pragmas at top:
- PRAGMA foreign_keys = ON;

Include indexes:
- idx_group_code ON dim_occupational_group(group_code)
- idx_effective_dates ON dim_occupational_group(effective_from, effective_to)
- idx_current_groups ON dim_occupational_group(effective_to) WHERE effective_to IS NULL
- idx_provenance_url ON scrape_provenance(url)
- idx_provenance_hash ON scrape_provenance(content_hash)

Include view:
- v_current_occupational_groups - JOIN groups with provenance WHERE effective_to IS NULL

2. **get_connection() function:**
   - Creates data/ directory if not exists
   - Returns sqlite3.Connection with:
     - row_factory = sqlite3.Row (dict-like access)
     - PRAGMA foreign_keys = ON (enforce FKs)
     - PRAGMA journal_mode = WAL (concurrent reads)

3. **init_db() function:**
   - Calls get_connection()
   - Reads schema.sql
   - Executes full schema with executescript()
   - Commits
   - Returns connection for chaining

4. **close_connection(conn) function:**
   - Safe close with None check

Use context manager pattern per RESEARCH.md:
```python
from contextlib import contextmanager

@contextmanager
def get_db():
    conn = get_connection()
    try:
        yield conn
    finally:
        conn.close()
```
