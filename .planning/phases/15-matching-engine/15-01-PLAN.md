  <name>Task 1: Create Pydantic output models for allocation results</name>
  <files>src/matching/__init__.py, src/matching/models.py</files>
  <action>
Create the matching package with Pydantic models capturing the full allocation result structure per CONTEXT.md decisions:

1. Create `src/matching/__init__.py` with lazy imports pattern (same as storage/__init__.py)

2. Create `src/matching/models.py` with these models:

**EvidenceSpan** - Single piece of evidence from JD:
- text: str (the quoted text)
- field: str (e.g., "Client-Service Results", "Key Activity 3")
- start_char: Optional[int] (position in original text, may be None if fuzzy matched)
- end_char: Optional[int]

**ReasoningStep** - Chain-of-thought reasoning step:
- step_number: int
- explanation: str (what this step determines)
- evidence: str (quoted text supporting this step)
- intermediate_conclusion: str

**GroupRecommendation** - Single group recommendation:
- group_code: str (e.g., "AI", "CS", "PM")
- group_id: int (FK to dim_occupational_group for provenance)
- confidence: float = Field(ge=0.0, le=1.0)
- confidence_breakdown: dict (definition_fit, inclusion_support, exclusion_clear, labels_boost, semantic_similarity)
- definition_fit_rationale: str (why this group's definition matches)
- reasoning_steps: List[ReasoningStep]
- evidence_spans: List[EvidenceSpan]
- inclusion_check: str ("none applies" or which inclusions support)
- exclusion_check: str (confirmation exclusions don't match)
- provenance_url: str (TBS source URL for this group)
- provenance_paragraph: Optional[str] (specific paragraph label)

**RejectedGroup** - Why a group was NOT recommended:
- group_code: str
- rejection_reason: str (why eliminated)
- exclusion_conflict: Optional[str] (which exclusion matched, if applicable)
- confidence_if_considered: float (what score would have been)

**AllocationResult** - Complete allocation output:
- primary_purpose_summary: str (extracted from JD)
- top_recommendations: List[GroupRecommendation] = Field(max_length=3)
- rejected_groups: List[RejectedGroup]
- borderline_flag: bool (True if top scores within 10%)
- match_context: str ("dominant match" | "competitive field" | "borderline - needs review" | "split duties detected")
- constraints_compliance: str = "Evaluated work described in JD, not person-specific attributes"
- duty_split: Optional[dict] (e.g., {"AS": 0.4, "PM": 0.35, "CS": 0.25} when split detected)
- warnings: List[str] (e.g., "Title mismatch: 'Clerk' title with PM-like duties")

Use Pydantic v2 syntax (model_validator, Field with description). Keep models simple for OpenAI structured output compatibility - avoid complex validation logic in models.

<task type="auto">
  <name>Task 2: Extend repository with inclusion/exclusion loading</name>
  <files>src/storage/repository.py</files>
  <action>
Add methods to OccupationalGroupRepository for loading groups with their statements:

1. **get_groups_with_statements()** - Load all current groups with inclusions/exclusions:
   - Query v_current_occupational_groups
   - For each group, query dim_occupational_inclusion and dim_occupational_exclusion
   - Return List[Dict] where each dict has: id, group_code, subgroup, definition, source_url, inclusions: List[str], exclusions: List[str]
   - Order inclusions/exclusions by order_num

2. **get_group_provenance(group_id: int)** - Get full provenance for a group:
   - Query scrape_provenance via source_provenance_id
   - Return dict with: url, scraped_at, content_hash, archive_path

3. **get_inclusions_for_group(group_id: int)** - Get inclusions only:
   - Return List[Dict] with: statement, order_num, paragraph_label

4. **get_exclusions_for_group(group_id: int)** - Get exclusions only:
   - Return List[Dict] with: statement, order_num, paragraph_label

All queries use parameterized SQL (? placeholders). Follow existing repository patterns.

The main method get_groups_with_statements() enables semantic matching to compare JD against complete group data including all TBS statements.
