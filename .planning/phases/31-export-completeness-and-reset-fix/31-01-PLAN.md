The reset() method must:
1. Restore `state` to `{ ...defaultState }` — use a shallow copy so the in-memory state is reset without mutating defaultState.
2. Remove the localStorage key: `localStorage.removeItem(STORAGE_KEY)` — wrapped in try/catch with console.warn on failure (match the pattern already used by `notify()`).

Resulting return object shape:
```javascript
return {
    getState: () => state,
    subscribe: (fn) => { ... },
    setState: (updates) => { ... },
    setSelections: (sectionId, newSelections) => { ... },
    reset: () => {
        state = { ...defaultState };
        try { localStorage.removeItem(STORAGE_KEY); } catch(e) { console.warn('localStorage remove failed:', e); }
        listeners.forEach(fn => fn(state));
    },
    notify
};
```

<task type="auto">
  <name>Task 2: Fix buildExportRequest() in export.js to handle 3 non-standard sections</name>
  <files>static/js/export.js</files>
  <action>
In `static/js/export.js`, fix the `buildExportRequest()` method (lines 45-130).

The current `Object.entries(state.selections).forEach(...)` loop (lines 58-80) does:
```javascript
const sectionData = profile[sectionId];
if (!sectionData || !sectionData.statements) return;  // silently skips 3 sections
```

This silently skips `core_competencies`, `abilities`, and `knowledge` because:
- `core_competencies`: lives at `profile.reference_attributes.core_competencies[]` as plain strings (not objects), NOT at `profile.core_competencies`
- `abilities`: items are inside `profile.skills.statements[]` filtered by `source_attribute === 'Abilities'`
- `knowledge`: items are inside `profile.skills.statements[]` filtered by `source_attribute === 'Knowledge'`

Replace the existing forEach loop body with special-case branches for these three sections. Keep the existing fields (id, text, jd_element, source_attribute, source_url, selected_at, description, proficiency) on all pushed objects. Use `now` for selected_at fallback (already defined above the loop).

CRITICAL jd_element and source_attribute values for the export payload — the backend uses these to route items to the correct PDF/DOCX section:
- core_competencies: `jd_element: 'core_competencies'`, `source_attribute: 'Core Competencies'`
- abilities: `jd_element: 'skills'`, `source_attribute: 'Abilities'`
- knowledge: `jd_element: 'skills'`, `source_attribute: 'Knowledge'`

For the standard path (key_activities, skills, effort, responsibility, working_conditions), keep the existing logic unchanged.

Replace the forEach callback body with:

```javascript
Object.entries(state.selections).forEach(([sectionId, selectedIds]) => {
  if (!selectedIds || selectedIds.length === 0) return;
  const now_ts = state.selectionTimestamps || {};

  if (sectionId === 'core_competencies') {
    // Plain strings at profile.reference_attributes.core_competencies[]
    const ccItems = profile.reference_attributes?.core_competencies || [];
    selectedIds.forEach(stmtId => {
      const index = parseInt(stmtId.split('-').pop(), 10);
      const text = ccItems[index];
      if (text !== undefined && text !== '') {
        selections.push({
          id: stmtId,
          text: text,
          jd_element: 'core_competencies',
          source_attribute: 'Core Competencies',
          source_url: null,
          selected_at: now_ts[stmtId] || now,
          description: null,
          proficiency: null
        });
      }
    });

  } else if (sectionId === 'abilities' || sectionId === 'knowledge') {
    // Filtered sub-array of profile.skills.statements by source_attribute
    const sourceAttr = sectionId === 'abilities' ? 'Abilities' : 'Knowledge';
    const filtered = (profile.skills?.statements || []).filter(s => s.source_attribute === sourceAttr);
    selectedIds.forEach(stmtId => {
      const index = parseInt(stmtId.split('-').pop(), 10);
      const stmt = filtered[index];
      if (stmt) {
        selections.push({
          id: stmtId,
          text: stmt.text,
          jd_element: 'skills',
          source_attribute: sourceAttr,
          source_url: stmt.source_url || null,
          selected_at: now_ts[stmtId] || now,
          description: stmt.description || null,
          proficiency: stmt.proficiency || null
        });
      }
    });

  } else {
    // Standard path: profile[sectionId].statements[]
    const sectionData = profile[sectionId];
    if (!sectionData || !sectionData.statements) return;
    selectedIds.forEach(stmtId => {
      const index = parseInt(stmtId.split('-').pop(), 10);
      const stmt = sectionData.statements[index];
      if (stmt) {
        selections.push({
          id: stmtId,
          text: stmt.text,
          jd_element: sectionId,
          source_attribute: stmt.source_attribute,
          source_url: stmt.source_url || null,
          selected_at: now_ts[stmtId] || now,
          description: stmt.description || null,
          proficiency: stmt.proficiency || null
        });
      }
    });
  }
});
```

Note: `state.selectionTimestamps` may be undefined — guard with `|| {}` as shown. Remove the `const now = new Date().toISOString();` that was previously defined inside the old forEach and keep it at the top of buildExportRequest() where it already exists (line 56).

No backend changes are required. pdf_generator.py, docx_generator.py, and export_service.py already handle `jd_element: 'core_competencies'` and `source_attribute: 'Abilities'/'Knowledge'` correctly.
